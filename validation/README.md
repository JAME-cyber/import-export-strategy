# 🎯 VALIDATION TERRAIN — Stop Building, Go Test

> Ce dossier existe pour **une seule raison** : zéro importateur réel n'a vu le Roster.
> Tant que ce dossier n'a pas produit 3 à 5 entretiens qualifiés, **aucune feature produit,
> aucun moat, aucun README n'est prioritaire**. C'est la règle.

---

## Pourquoi ce dossier avant tout le reste

La vidéo Ogma (Damien Würsten) a vendu **450 k€ en 3 mois** — parce qu'il est allé parler à
des gens qui avaient une douleur réelle. Il n'a pas peaufiné de moat avant.

Nous avons l'inverse : un produit construit, zéro test terrain. **Le risque #1 du projet n'est
pas la commoditisation (Ogma). C'est de ne pas savoir si le pain existe à notre échelle.**

## Critère de réussite — défini À L'AVANCE (anti-biais)

On ne se mentira pas à soi-même. Le verdict se lit sur **le comportement passé**, pas sur les
 compliments en fin d'entretien.

| Signal | Critère | Décision |
|---|---|---|
| 🟢 **FORT** | ≥ 2/5 prospect raconte une **perte chiffrée vécue** + a **déjà payé** pour un outil/formation transitaire + après démo pose une **question d'achat** (« comment on accède », « ça coûte combien ») | → Construire l'accès. Pricing réel à tester. |
| 🟡 **MOYEN** | Douleur ressentie (anecdotes) mais **jamais payé** pour rien | → Pain réel mais non monétisé. Tester pricing agressif / freemium. |
| 🔴 **FAIBLE** | « Ça a l'air sympa » mais aucune question concrète, aucune perte vécue | → Pas un pain. Pivot de segment ou d'angle. Ne pas construire plus. |

## La règle anti-triche

**Ne jamais demander « paieriez-vous ? »** dans l'entretien. Tout le monde dit oui pour être
sympa. Le vrai signal c'est : *ont-ils déjà payé pour un truc similaire ?* (question 7) +
*posent-ils une question d'achat après la démo ?* (observation question 10).
